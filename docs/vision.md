# Vision

## Product

AI Mantra Generator.

## Goal

Create a prototype where a user can describe what they need, configure mantra parameters, and move into a playground experience for processing.

## Current Prototype Scope

- Intake page with hero, prompt input, settings, quick prompts, and processing action.
- Playground page after processing.
- Components page as UI sandbox.
- Grid overlay for visual alignment.

## Product Direction

The product should support a spec-driven workflow:

```text
Idea
-> Requirements
-> UX/UI
-> Frontend prototype
-> Processing experience
-> Audio generation
-> Export/share
```

## Quality Bar

- The UI should feel intentional, not generated by accident.
- Components should be reusable.
- Interaction states should be explicit.
- Each feature should be verified before the next one is added.
